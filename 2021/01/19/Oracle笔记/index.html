10.视图

实际上就是临时表，保存的是select语句。每次从视图中查询数据，都是动态的执行视图的select语句。

1.简化查询语句，比如部门的平均薪资，如果将平均薪资保存为视图，那么查询就只需要从视图中查

2.数据保护。平台之间的数据库交互，只提供需要的数据给外部系统，而不是提供所有的基础数据。比如部门的平均薪资需要共享，那么就只需要共享这个视图就可以了，而不是共享所有的数据。

语法:
create [or replace] view v_name as (select 语句)

1.简化select语句的编写;

2.数据保护：

只提供了需要提供的数据，没有暴露其他数据

使用sys用户给scott用户赋权限
使用scott用户查询

思考：视图是否可以修改？update更新数据的语句？？？

肯定是不可以的，所有的修改最终都会体现到基础表中，违背了基础表的修改原则。不违背基础表规则的情况下，可以修改

但是！！！！！一般视图，不做修改！！！！！！！视图是从基础表中提取出来的，修改也是修改基础表。视图的查询是实时的，基础表数据更新，视图查询也会实时更新。

11.索引

作用：快速定位，加快查询的速度。
类似新华字典的检索
比如小说中的目录章节—》快速定位

怎么看sql语句有没有走索引？

sqlDeveloper F10
plSql F5

select语句走索引的基本条件

1）列有创建索引

2）有where条件，并且where条件后的列，创建了索引

索引的创建

1）主键，唯一键是会自动创建索引

2）单列索引

3）组合索引

4）函数索引

索引是不是越多越好？

不是

1.索引是占用内存空间的，是存在的，也占用物理空间。

2.数据库表如果有更新，而且涉及到索引，那么相应的索引也需要做同步更新

什么样的列适合创建索引？

值越唯一，越适合创建索引，能快速定位。比如性别这样的列，就不适合，不能做到快速定位

索引无效的场景：

1.使用like，并且% _ 打头

2.隐式的类型转换

3.使用 != 判断，不走索引

12.序列

类似mysql中的自增长序列，但是oracle中的序列是一个独立的对象。并且序列的步长可以随意调整

语法：

create sequence 序列名称

完整语法：

create sequence SEQ_TEST  
minvalue 1   --最小值
maxvalue 999999999999999999999999999   --最大值
start with 1  --开始值
increment by 1  --步长
cache 20;     --缓冲区大小

缓冲区：oracle会从序列中提前生成数字放入缓冲区，程序从序列中获取数字时，直接从缓冲区获取。

序列这个对象有两个属性

nextval   下一个序列值
currval    当前序列值（序列创建之后，必须先nextval取值，否则没有当前值）

序列用于insert语句

序列本质上和表是独立的，这两者没有关系。但是从业务逻辑上来说！！！！序列和表又是 一 一对应的！！是一对一的关系。

declare

  --定义自定义对象类型
  type EmpInfo is record(
    v_empno     emp.empno%type,
    v_ename     emp.ename%type,
    v_sal       emp.sal%type,
    v_sal_grade salgrade.grade%type,
    v_dname     dept.dname%type);
  --定义变量v_emp,是自定义对象EmpInfo类型
  v_emp EmpInfo;
begin
  select e.empno, e.ename, e.sal, s.grade, d.dname
    into v_emp  --查询结果集必须和EmpInfo中的属性的定义顺序一致
    --into v_emp.v_empno,
    --     v_emp.v_ename,
    --     v_emp.v_sal,
    --     v_emp.v_sal_grade,
    --     v_emp.v_dname
    from emp e
    left join salgrade s
      on e.sal between s.losal and s.hisal
    left join dept d
      on e.deptno = d.deptno
   where e.empno = v_no;

   dbms_output.put_line(v_emp.v_ename || ',' 
                        || v_emp.v_sal || ','
                        || v_emp.v_sal_grade || ','
                        || v_emp.v_dname || ',');
end;

10）游标

pl/sql编程中的查询结果集可以看做是游标。或者说，查询结果集可以使用游标来接收

游标定义

1）游标的定义
cursor 游标名称 is 
    select 查询语句

2）定义一个变量，接收游标中的行数据

3）打开游标

4）从游标中取值

5）关闭游标，释放内存

游标定义、使用的完整代码:

declare 
  --1.定义游标
  cursor c is 
         select ename,sal from emp;

  --2.定义变量，用于接收游标中的行数据
  --从游标中取值时需要使用
  --变量类型  游标的行对象类型
  v_c c%rowtype;
begin
  -- 3.游标打开，加载数据到内存中
  open c;

  -- 4.从游标中取值
  --fetch，指针往下移一位，将指针指向的数据取出，赋值给v_c变量
  fetch c into v_c;
  --v_c是一个行对象类型，取值需要告知对象的具体属性（或者说列名）
  dbms_output.put_line(v_c.ename || ',' || v_c.sal);

  --再次从游标中取值
  fetch c into v_c;
  dbms_output.put_line(v_c.ename);

  fetch c into v_c;
  dbms_output.put_line(v_c.ename);

  -- 5.关闭游标，释放内存
  close c;

end;

动态游标
游标在打开的时候动态指定select语句

1)定义游标类型

变量定义

4）批量取值

代码：
– 动态游标

declare 
  -- 定义一个类型，这个类型是游标类型
  type v_cursor is ref cursor;

  --定义一个变量，是v_cursor类型
  c  v_cursor;


  --定义一个变量，接收游标中的数据
  --v_c c%rowtype;  不能使用游标的行对象类型,因为c自己都不知道有什么列
  v_ename emp.ename%type;
  v_empno emp.empno%type;


  --用于批量取值的变量
  type Enames is table of emp.ename%type index by binary_integer;
  v_enames Enames;

begin
  -- 打开动态游标，在打开时，给定select语句
  open c for select ename,empno from emp;

  --取值
  loop
    --赋值的变量顺序，需要和游标的查询结果集一致
    fetch c into v_ename,v_empno;
    exit when c%notfound;
    dbms_output.put_line(v_ename || ',' || v_empno);
  end loop;

  close c;

  --批量取值
  dbms_output.put_line('---------批量--------------');
  open c for select ename from emp;

  loop

  dbms_output.put_line('-------');


  --bulk collect 批量取值的关键字
  --limit 5  表示一次取5条数据
   fetch c bulk collect into v_enames  limit 5;   

   for v_index in 1..v_enames.count loop
     dbms_output.put_line(v_index ||',' ||v_enames(v_index));
   end loop;

   --批量取值，退出条件的位置和单个取值时不一致
   exit when c%notfound;

   end loop;

  close c;

end;

案例
–给部门薪资最低的所有的员工加薪100

create or replace procedure p_update_sal is
  --变量，存放部门编号
  v_dno emp.deptno%type;

  v_sql varchar2(100);
begin

  --查询平均薪资最低的部门编号
  select deptno
    into v_dno
    from v_dept_avg
   where avg_sal = (select min(v_dept_avg.avg_sal) from v_dept_avg);

  --数据备份
  v_sql := 'create table emp_bak'|| to_char(sysdate,'yyyymmddhh24miss') || ' as select * from emp';

  dbms_output.put_line(v_sql);
  --使用命令执行sql
  execute immediate v_sql;

  --给这个部门下的员工加薪100  update
  update emp set sal = sal+100 where emp.deptno = v_dno;

  --需要手动提交数据
  commit;
end;
